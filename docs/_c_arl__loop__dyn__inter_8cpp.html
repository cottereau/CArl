Detailed Description

Program responsible to calculate \( rhs_{inter} = - ( C^A\dot{U}^A_{free} (t)+C^B \dot{U}^B_{free} (t)) \). This program's input file description can be found at the documentation of the function carl::get_input_params(GetPot& field_parser, feti_loop_dyn_params& input_params).

Required parameters:

  • Scheduler parameters:
    • ClusterSchedulerType : the cluster scheduler used in your system, e.g. SLURM
    • ScriptFile : Path to the base SLURM script file
  • FETI solver parameters:
    • ScratchFolderPath : Path to the scratch folder
    • ResultFolderPath : Path to the result folder
    • TildeMatrixA : Path to the system matrix of A, i.e. \( \tilde{M}^{A} \)
    • CouplingMatrixA: Path to the coupling matrix of A, i.e. \( C^{A} \)
    • StiffnessMatrixA : Path to the stiffness matrix of A, i.e. \( K^{A} \)
    • RHSVectorA: Path to the force vector of A, i.e. \( F^{A}_{base} \)
    • ForceFolderA: Path in which all of the forces will be stored, i.e. \( F^{A}(n\Delta T)=F^{A}_{base}\times f(n \Delta T) \)
    • TildeMatrixB : Path to the system matrix of B, i.e. \( \tilde{M}^{B} \)
    • CouplingMatrixB : Path to the coupling matrix of B, i.e. \( C^{B} \)
    • StiffnessMatrixB : Path to the stiffness matrix of B, i.e. \( K^{B} \)
    • RHSVectorB : Path to the force vector of B, i.e. \( F^{B}_{base} \)
    • ForceFolderB : Path in which all of the forces will be stored, i.e. \( F^{B}(n\Delta t)=F^{B}_{base}\times f(n \Delta t) \)
    • InterpolationMatrix : Path to the interpolation matrix, i.e. \( H \)
    • InnerSolverInputGeneral: Path of the input file to CArl_loop_dyn_setup.cpp, which will be served as general extry
  • External solver parameters
    • ExtSolverInputA : Path to a general external solver input of object A
    • ExtSolverInputB : Path to a general external solver input of object B
    • ExtSolverInputGeneral : Path to an external solver input of interpolation matrix
    • ExtSolverLaunchScript : Script to launch external solver, e.g. 'srun -n 4 $CARLBUILD/libmesh_solve_linear_system -i '
    • GeneralEntryFilePath : Path of the input file to CArl_loop_dyn_setup.cpp, which will be served as general extry

  • Newmark parameters

    • deltatA : \( \Delta T_A \), time step of A, in seconds
    • betaA : \( \beta_A \) for Newmark method in displacement of A
    • gammaA : \( \gamma_A \) for Newmark method in speed of A
    • deltatB : \( \Delta t_B \), time step of B, in seconds
    • betaB : \( \beta_B \) for Newmark method in displacement of B
    • gammaB : \( \gamma_B \) for Newmark method in speed of B
    • SimulationDuration : total duration of simulation, in seconds

    Attention: SimulationDuration should be integer multiple of deltatA. deltatA should be integer multiple of deltatB.

In this step, it will use the following files in scratch foler:

  • prev_acc_A.petscvec : \( \ddot{U}^A(T-\Delta T) \)
  • this_acc_A_free_sys_sol_vec.petscvec : \( \ddot{U}^A(T) \)
  • this_acc_B_free_sys_sol_vec.petscvec : \( \ddot{U}^B(t) \)

It will create:

  • inter_acc_A_free.petscvec : \( \ddot{U}^A(t) \)
  • rhs_interpolation_vec.petscvec : \( rhs_{inter}(t) \)
